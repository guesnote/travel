# 旅行管理工具｜部署說明

## 📁 檔案說明

| 檔案 | 說明 |
|------|------|
| `index.html` | 前端主程式，可直接用瀏覽器開啟 |
| `Code.gs` | Google Apps Script 後端，負責與 Google Sheets 同步 |

---

## 🚀 部署步驟

### Step 1：建立 Google Sheets

1. 前往 [Google Sheets](https://sheets.google.com) 建立新試算表
2. 記下試算表網址（之後會用到）

### Step 2：建立 Apps Script

1. 在試算表中，點選上方選單「**擴充功能**」→「**Apps Script**」
2. 刪除預設的空白內容
3. 將 `Code.gs` 的內容完整貼上
4. 點選儲存（Ctrl+S）

### Step 3：初始化工作表

1. 在 Apps Script 編輯器中，選擇函式 `initSheets`
2. 點選「**執行**」按鈕
3. 第一次執行需授予權限，依提示允許即可
4. 完成後 Sheets 會自動建立 4 個工作表：`Trip` / `Wallet` / `Lists` / `Settings`

### Step 4：部署為網頁應用程式

1. 點選右上角「**部署**」→「**新增部署作業**」
2. 類型選擇「**網頁應用程式**」
3. 設定如下：
   - 執行身分：**我**
   - 存取權：**知道連結的任何人**（需登入 Google 帳戶）
4. 點選「**部署**」
5. 複製產生的「**網頁應用程式網址**」

### Step 5：連結前端

1. 用瀏覽器開啟 `index.html`
2. 切換到「**設定**」分頁（右下角齒輪圖示）
3. 在「Apps Script 網址」欄位貼上剛才複製的網址
4. 點選「**立即同步**」測試連線

---

## ⚠️ 重要注意事項

**每次重新部署後網址會改變！**

更新 `Code.gs` 後若重新部署，需要：
1. 複製新的部署網址
2. 在 `index.html` 設定頁更新網址

建議：修改 Code.gs 後，使用「**管理部署作業**」→「**編輯**」現有版本，而非建立新部署，這樣網址不會改變。

## 💾 資料架構

### Google Sheets 工作表

| 工作表 | 儲存內容 |
|--------|---------|
| `Settings` | 旅程名稱、日期、匯率、常用連結 |
| `Trip` | 每日行程（JSON） |
| `Wallet` | 消費記錄（JSON） |
| `Lists` | 必備物品、必買清單、備忘錄 |

### 同步策略

```
開啟 App     → 自動從 Sheets 拉取最新資料 (Pull)
操作過程中   → 即時存 LocalStorage（離線可用）
每 3 分鐘    → 自動推送至 Sheets (Push)
手動同步鈕   → 隨時強制同步
```

---

## 📱 使用建議

- **出發前**：在電腦瀏覽器上編輯行程，完成後點「立即同步」
- **旅行中**：手機開啟後自動載入最新資料，記帳時離線也可用
- **換裝置**：只要連同一個 Apps Script 網址，資料自動同步
- **備份**：設定頁有「匯出 JSON」功能，可定期備份
